public subroutine get_band_energy()

Read the band file info which is prerequisite to know about k-points bands etc. It would be nice for od2od to work this out for itself. But at least this way it is consistent. The problem is that to convert, say a ome to a ome one also requires a .bands file.

public subroutine pad_an_ome()

This routine takes OptaDOS's internal representation of an dome file, and uses it to construct an ome file, where the off diagonal elements are padded with zeros. It might be helpful if we're reading in other codes' input files. It might also be useful for testing OptaDOS itself.
